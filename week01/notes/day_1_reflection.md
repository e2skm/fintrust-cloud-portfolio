# Day 1 Reflection

## Programme Foundation

Today I gained a clear understanding of the structure of the 16-week Cloud to Solutions Accelerator programme and how the four learning streams—Cloud & AWS, SQL & Data Engineering, Python Programming, and Solutions Architecture—work together to prepare us for the AWS Solutions Architect Associate (SAA-C03) certification. I also learned about the FinTrust Bank capstone scenario, which will be used throughout the programme to apply cloud concepts to a realistic business case.

## LMS and GitHub Setup

I successfully learned how to navigate the Edusignis LMS, locate daily learning resources, complete knowledge checks, and understand the weekly portfolio submission process. I also learned the importance of maintaining a professional GitHub portfolio and how to structure repositories, folders, and commits for ongoing programme work.

## Amazon Cloud Quest Experience

The afternoon session introduced Amazon Cloud Quest, a game-based learning platform that provides hands-on experience with AWS services in a safe sandbox environment. I created or accessed my AWS Skills Builder account, explored the Cloud Quest interface, and became familiar with the AWS Management Console. One key takeaway was understanding how AWS services are organised into categories such as Compute, Storage, and Database.

## Challenges and Areas for Improvement

The most confusing part of the day was navigating the different tools and understanding how they all connect to the overall programme journey. Exploring the Cloud Quest interface helped reduce this confusion, and the Learning Briefs provided useful context before entering the Practice Labs.

## FinTrust Connection

Today's activities showed how AWS can help organisations like FinTrust Bank move away from traditional on-premises infrastructure toward scalable cloud-based solutions. The AWS console and services introduced today are the same tools architects use to design and deploy modern cloud systems.

## Question for Tomorrow

I would like to better understand how AWS Regions, Availability Zones, and Edge Locations work together to provide high availability, resilience, and low-latency services for customers across different locations.
